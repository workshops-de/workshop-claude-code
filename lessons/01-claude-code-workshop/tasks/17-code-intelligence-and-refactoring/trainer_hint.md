## Learning goals

- **Skills:** directing intent-driven refactors; protecting behaviour with tests; tool selection.
- **Concepts:** refactoring vs. rewriting; modularity and readability; behaviour preservation.
- **Takeaways:** delegate judgement-heavy refactors; keep mechanical edits in the IDE; verify with
  tests.

## Facilitation notes

- This opens "Quality & Automation" — set the frame that the rest of the morning is about making
  quality systematic and scriptable.
- If your organization has an LSP plugin available, a live before/after (grep-based vs.
  code-intelligence-based refactor) is a strong demo.
