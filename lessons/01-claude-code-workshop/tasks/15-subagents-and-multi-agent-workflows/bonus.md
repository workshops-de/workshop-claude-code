## Going further — multi-agent at scale

A single subagent is the start. Claude Code can coordinate **many** agents:

- **Run agents in parallel** — subagents, agent view, agent teams, and dynamic workflows.
- **Agent teams** — multiple Claude Code sessions working together with shared tasks and
  inter-agent messaging.
- **Agent view** — dispatch and manage many sessions from one screen.
- **Dynamic workflows** — Claude writes a rerunnable script that orchestrates many subagents for
  audits and large migrations.
- **Worktrees** — isolate parallel sessions and subagents so their changes don't collide.

Try running your explorer subagent and a second "test runner" subagent in parallel.
