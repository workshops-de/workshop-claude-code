## Learning goals

- **Skills:** launching and steering Claude Code; reading agent activity; using planning mode;
  monitoring context usage; managing permissions.
- **Concepts:** the agent control loop (Reason → Act → Observe); tools vs. text; short-term vs.
  long-term memory; context as the primary resource.
- **Takeaways:** explore before you edit; plan before you act; you remain the owner of every
  change.

## Facilitation notes

- Confirm everyone finished the installation task before starting this one — a broken install here
  wastes far more time than in the dedicated setup task.
- Participants sometimes jump straight to asking for code. Redirect them: the point of this task
  is fluency with the tool and the loop, not output.
- Watch for participants skipping planning mode entirely — this is the single most valuable habit
  to instill early, and it pays off for the rest of the workshop.
- Common issue: participants confuse "the agent didn't do much" with "the agent is bad" when the
  real cause is an empty/thin context. Use this as the first live example of "context is king".

## Time estimate

~30 minutes is enough for an experienced audience; add 10–15 minutes if this is most
participants' first contact with an agentic CLI tool.

## Discussion points

- Ask a few participants to describe what happened in each phase of the Reason → Act → Observe
  loop during their session.
- Ask where the agent asked for permission and whether they would have granted it in a real
  project.
